Sprint 3 - Finalisation et Scan (SCAN TERMINE)

Contexte

L’operateur a saisi la demande (Sprint 1) et a eventuellement gere un duplicata (Sprint 2).
Il possede maintenant les documents physiques du demandeur.
Il doit les numeriser et les rattacher a chaque piece justificative attendue.

Une fois toutes les pieces scannees, l’operateur verrouille le dossier : plus aucune modification possible sur la demande, le demandeur ou le passeport.
Le statut de la demande passe alors de CREE ou "demande creee" a SCAN TERMINE.

Objectif

* Ajouter la gestion des fichiers scannes (upload) par piece justificative
* Controler la completude : bouton "SCAN TERMINE" actif uniquement quand toutes les pieces attendues (cochees dans demande_piece) ont un fichier associe
* Verrouillage definitif des donnees apres clic sur "SCAN TERMINE"
* Nouveau statut SCAN TERMINE dans la table statut_demande

Sprint 4 - Scan Qr code
Contexte: 
On veut une application frontend (Vue.js) qui permet de suivre les demandes d’un utilisateur en entrant soit un numéro de demande, soit un numéro de passeport. L’application appelle une API REST développée en Spring Boot.
Si l’utilisateur entre un numéro de passeport, toutes ses demandes s’affichent avec leurs statuts, triées par ordre chronologique (la plus récente en premier). Si c’est un numéro de demande, cette demande apparaît en premier, suivie des autres demandes liées.
Chaque demande doit afficher un historique de statuts (ex : dossier créé, scan terminé, etc.), pour permettre un suivi complet.
Lors de la création d’une demande, un QR code est généré contenant une URL. En scannant ce QR code avec un téléphone, l’utilisateur est redirigé vers l’application qui affiche directement le suivi de ses demandes via l’API.
L’application peut être accessible en localhost ou via l’adresse IP du PC pour un accès mobile.

en gros :
1. Spring Boot (backoffice) → expose les endpoints API
2. Vue.js  → consomme ces endpoints + reçoit les redirections QR
3. framework(frontoffice )→ on change juste l'URL du QR code pour pointer vers Vue.js
